# BookMeet

Calendly-like tool for booking recurring 30-min weekly meetings (5 weeks). Admin creates scheduling links, customers book via public link.

## Stack

- **Frontend**: React + Vite + TypeScript + Tailwind CSS v4 (Vite plugin, no PostCSS)
- **Backend**: Convex (schema, queries, mutations, actions, HTTP routes)
- **Calendar**: Google Calendar API via `googleapis` in Convex Node actions (`"use node"`)
- **Auth**: Google OAuth (admin only, single user: calvin@airops.com)

## Commands

- `npm run dev` — start Vite dev server (localhost:5173)
- `npx convex dev` — start Convex dev server (watches and deploys backend)
- `npm run build` — typecheck + production build

Both must run simultaneously during development.

## Project Structure

```
convex/              # Backend
  schema.ts          # Tables: schedulingLinks, bookings, adminTokens, availabilityCache
  http.ts            # Google OAuth redirect + callback endpoints
  adminTokens.ts     # Internal token storage/refresh
  googleCalendar.ts  # "use node" — freebusy queries + recurring event creation
  availability.ts    # Public getSlots action with 5-min cache
  schedulingLinks.ts # CRUD + internal query
  bookings.ts        # Booking action + queries
src/
  components/ui/     # Button, Card, Badge, Input, Select
  components/layout/ # AdminLayout (with session guard), Sidebar
  components/calendar/ # CalendarGrid, TimeSlotList, TimezoneSelector
  components/booking/  # BookingForm
  pages/admin/       # Login, Dashboard, CreateLink
  pages/booking/     # BookingPage, Confirmation
  lib/utils.ts       # cn(), date formatting, timezone helpers
```

## Key Architecture Decisions

- Availability check queries Google freebusy for admin + optional co-host across all 5 weekly dates; a slot must be free on ALL 5 weeks
- Co-host freebusy works within Google Workspace without separate auth
- OAuth tokens stored in Convex DB (server-side only); access tokens auto-refresh
- Each scheduling link is single-use (status goes active → booked)
- Booking action re-verifies availability before creating the calendar event

## Environment Variables
