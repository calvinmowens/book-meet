# BookMeet

Calendly-like tool for booking recurring meetings. Admin creates scheduling links, customers book via public link.

## Stack

- **Frontend**: React + Vite + TypeScript + Tailwind CSS v4 (Vite plugin, no PostCSS)
- **Backend**: Convex (schema, queries, mutations, actions, HTTP routes)
- **Calendar**: Google Calendar API via `googleapis` in Convex Node actions (`"use node"`)
- **Email**: Gmail API for branded booking notifications
- **Auth**: Google OAuth (admin only, single user: calvin@airops.com)

## Meeting Types

**Weekly Sync** — 30 min, 5 weekly meetings on one day. Single time selection.

**Cohort** — 60 min, 3-week program (2/2/1 structure):
- Week 1-2: 2 meetings on M/W or T/Th
- Week 3: 1 concluding meeting on W or Th
- Customer picks track (M/W or T/Th), start week, and separate times for each day
- Events created individually (no RRULE), numbered 1/5 through 5/5

## Commands

- `npm run dev` — Vite dev server (localhost:5173)
- `npx convex dev` — Convex dev server (watches + deploys backend)
- `npm run build` — typecheck + production build

Both must run simultaneously during dev.

## Key Architecture Decisions

- Admin calendar: events.list API (distinguishes OOO from real conflicts)
- Co-host calendar: freebusy API (works cross-Workspace without sharing)
- OOO events ignored for availability; affected sessions skipped (not rescheduled)
- Weekly sync: RRULE with EXDATE for OOO weeks, COUNT extended to compensate
- Cohort: individual events created per session (different times per day group)
- Links support re-booking (status active → booked, booked links show confirmation prompt)
- Link lifecycle: active → booked → archived (archive/reactivate/delete)
- Branded HTML email notification sent to admin via Gmail API on each booking
- Event title format: `AirOps <> {Client}, Weekly Sync` or `AirOps <> {Client}, Cohort (N/5)`
- Conference solution auto-detected (Zoom add-on preferred, falls back to Google Meet)
